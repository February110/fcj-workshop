### Overall Evaluation

The First Cloud Journey program gave me a practical learning path for cloud computing. The program did not only focus on reading documentation or following labs, but also required me to apply what I learned to a final team project. This made the internship more realistic and helped me understand how AWS services are used in an actual application.

### Learning Experience

The AWS labs were useful for building foundation knowledge. I started with basic services such as IAM, VPC, EC2, S3, and DynamoDB, then moved to services related to the PeriodIQ project such as Lambda, API Gateway, CloudWatch, SQS, and frontend deployment on S3/CloudFront.

The most valuable part was applying those services to PeriodIQ. In my role as **Person 2 - Rule Engine & Workout Generation**, I had to think beyond individual services and understand the complete flow: user input, API request, Lambda processing, DynamoDB output, logs, and the final user interface.

### Mentor and Team Support

The mentor and support team provided helpful guidance when I had questions about AWS services, report structure, and project implementation. The guidance helped me avoid only copying lab steps and instead focus on explaining why each service was used in my project.

Teamwork was also important because the PeriodIQ project had multiple roles. My work depended on inputs from the authentication flow and also needed to provide output that other flows could use, such as `WorkoutPlan` data and asynchronous messages.

### What I Found Most Useful

* Hands-on AWS labs helped me understand cloud concepts faster.
* The final project made the learning process more realistic.
* The report and workshop structure helped me organize technical work clearly.
* The live PeriodIQ deployment gave me a concrete result to verify and present.

### Suggestions

* Provide a clearer checklist for final project evidence, such as required screenshots, logs, and architecture diagrams.
* Add more examples showing how to convert AWS lab knowledge into project documentation.
* Encourage teams to define integration contracts earlier, especially API payloads and shared database schemas.
* Keep the final project requirement because it helps interns connect AWS theory with real implementation.

### Final Feedback

Overall, this internship was useful because it combined learning, implementation, teamwork, and documentation. After completing the program, I have a better understanding of AWS serverless architecture and how to present a cloud project clearly.
